显示源码pre/xmp

                    pre：展示非html源码。有html代码，仍会被翻译
                    xmp:展示任何源码。即使html的源码也不翻译，直接展示

js中的{ //在php中不能解析问题

在html中写 { //通过页面的物理路径访问没有错 } 但是在thinkphp3.2中通过控制器调用。{ //没法解析出来，导致页面展示会报错 建议写法改为 { //注释 代码 }
1、当前仅在tp3.2中遇到。其他php有待考证,初步怀疑是tp模板替换的bug。
2、此处的{ //中间的空格是人为加到。不然解析不出来，看不到
